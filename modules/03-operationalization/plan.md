# Modul 3 — Operationalization

**Ziel:** Guten Kontext über Requests und Projekte hinweg persistent machen. Organisiert nach *Zweck*, entlang der Frage, die Entwickler wirklich stellen: „Wie bringe ich den Agent dazu …?"

## Foliennamen (in Reihenfolge)

1. Guten Kontext persistent machen — über Requests und Projekte hinweg
2. Die Frage des Entwicklers: „Wie bringe ich den Agent dazu …?"
3. Drei Gruppen von Artefakten: Wissen, Workflows, Constraints
4. Wissen → `CLAUDE.md`, `AGENTS.md`: Abwesenheiten, Konventionen, Pointer
5. Wiederverwendbare Workflows → Skills, Slash Commands
6. Constraints → Rules, Guidelines, Hooks, Tests
7. Always-on vs. on-demand vs. off-context
8. Instructions für den Arbeitsblock
